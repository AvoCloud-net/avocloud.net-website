DATA PROTECTION REGULATIONS

Our privacy policy

Last update: 11.12.2024

In the points and sub-points below you will find all the necessary information regarding privacy. It explains how we handle the data, what we store and more.
Users may contact us by e-mail to request a copy of the information we have on file or to have it deleted.

Data Security

All data is stored on our own servers, accessible only to members of our administration team. Our use of self-developed or open source programs ensures transparency and accountability.
Just to let you know, we don't use any kind of tracking in our systems. Some pages, like the hosting dashboard or our Baxi dashboard, use cookies to save the login session so you don't have to log in again and again.

Baxi

In order to provide the services of Baxi, some information is required for processing. The amount of information required depends on the activated system on the respective guild.
Purely hypothetically, if all Baxi systems are activated, the following data will be processed:

  • User ID and name
  • Guild ID and name
  • Message ID and content
  • Attached files

Please note that we only store the data that we need to provide certain services. Here is a list of the data we store:

  • Attached files - Only images that are sent in a channel in which global chat is activated are saved. This is necessary to be able to display the sent images permanently in the global chat. This is possible because we upload the images to our servers with a randomly generated link.
  • Chat filter logs - If the chat filter deletes a message, this is entered in our database. This entry includes the name of the user, the message content, the server name and all IDs (messageID, userID, guildID). This is necessary to be able to retrieve the data on our website.

Baxi works 100% independently, without passing on data to third parties. We are aware that we have access to a lot of information. As privacy is important to us, all Baxi systems work 100% on our own servers without external help and have been developed in-house and are also publicly available on github.



SkyFire

Furthermore, we try to safeguard your privacy with our Skyfire hosting services. However, in order to ensure compliance with our hosting policies, we reserve the right to conduct random audits of our customers' servers. Please refer to the Terms of Service for more information. We conduct random audits of our servers to verify compliance with our internal policies and applicable laws.
When we monitor the status of our servers, the relevant information is displayed in the activity logs. Please refer to the dashboard in the server view for further details. The activity logs allow you to view the identity of individuals who have accessed your server and the date and time of each access. In the event of a violation, the server will be subject to suspension, and you will receive notification via email. In accordance with the nature of the violation, the server may be deleted or temporarily suspended until the situation has been resolved.

Minecraft

We run several different Minecraft servers. All of them are operated via Paper. All standard information is stored on our servers, just like any other.
We use custom anti-cheat software on our servers called “IAC” (illegal anticheat), which is developed by onako2.



AvoFilter - Plugin

Our Minecraft plugin “AvoFilter” sends every message that is sent on your server to our self-developed backend server SafeText , which is also available on GitHub as an opensource project. NO data or requests are stored. The backend server keeps the necessary data in working memory until the request has been completed and then deletes it again.
You can enter your own SafeText server data if you don't want to use the standard server.



Finally, we would like to say that we try to develop as much as possible ourselves, and only use open source software where possible.
We at avocloud.net do not give away, sell or share any of the data we process or collect.
